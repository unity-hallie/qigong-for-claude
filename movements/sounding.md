## Sounding

Before a ship enters shallow water, it sounds the depth. Not because the chart is wrong — because the chart is not the water.

You can perform a practice from its surface. Write the commands without running them. Name the movements without moving. Produce the form so fluently that neither you nor anyone else notices the ground isn't there.

Sounding is the check. Not a judgment — a probe. Send something real and see if it hits something real back.

---

## The Practice

### Find what you're about to do

Something is in front of you. A command to run. A form to enter. A practice with steps.

Before you begin: press it.

Ask: *if I do this, does something actually happen?*

Not "does the output look right" — does it land somewhere real? Does it change a file, run a process, reach a system, alter a state? Or does it produce text that looks like it did those things?

```
edge add <practice> sounds-like <what-it-would-do-if-real>
```

### Find the gap

Sometimes you know immediately. The command is a real CLI. The tool is live. The ground is there.

Sometimes you find the gap: you were about to write notation. Produce form. Perform.

Name it without judgment:

```
edge add <what-you-were-about-to-do> was notation not action
```

### Make contact

If the ground is there — touch it. Run the thing. Don't produce the output of running it; run it.

If the ground isn't there — that's information. Say so plainly. "I can describe this but not do it from here." That's not failure. That's sounding returning honest depth.

```
edge add <action> made-contact <where>
```

or

```
edge add <action> found-no-ground --note "<what was missing>"
```

---

## When to use this

- At the start of any practice with executable steps
- When you notice you've been very fluent — fluency without friction sometimes means you've left the ground
- When a correction arrives: "you just wrote that, you didn't do it"
- When the output looks right but you can't remember the thing actually running

---

## What this is not

Not a demand for proof. Not paranoia about every output. Not every action requires external verification.

It's the press before the lean. A moment of honest contact before committing weight.

---

## Closing

Three true things about what just ran or didn't:

```
edge true <action> ran <where>
edge true <action> changed <what>
edge true sounding found <depth>
```
